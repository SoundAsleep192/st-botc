# Изнанка Хокинса — симулятор партий

Bun + TypeScript. Гоняет N партий выбранного состава с разным AI-поведением и считает win-rate'ы для анализа баланса.

## Установка

```bash
cd sim
bun install
```

## Запуск

```bash
# 1000 партий на 10 игроков, демон — Векна, seed фиксирован
bun run sim --players 10 --games 1000 --demon vecna --seed 42

# все демоны по очереди, 500 партий каждого
bun run sim --players 13 --games 500 --demon all

# подробный лог одной партии
bun run sim --players 10 --games 1 --demon whatsit --verbose
```

### Флаги

| Флаг         | Значение по умолчанию                         | Описание                                          |
|--------------|-----------------------------------------------|---------------------------------------------------|
| `--players`  | `10`                                          | Размер партии: 7, 10, 13, 15                      |
| `--games`    | `1000`                                        | Сколько партий прогнать                           |
| `--demon`    | `vecna`                                       | `vecna` / `mindflayer` / `whatsit` / `all`        |
| `--seed`     | случайный                                     | Сид PRNG для воспроизводимости                    |
| `--ai`       | `simple`                                      | `simple` (случайные цели) / `smart` (позже)       |
| `--verbose`  | `false`                                       | Печатать ход каждой партии                        |

## Архитектура

```
src/
├── types.ts              Глобальные типы: Role, Player, GameState, Event
├── rng.ts                Seedable PRNG
├── config.ts             Раскладки по размеру партии
├── roles/
│   ├── registry.ts       Реестр всех 25 ролей (метаданные + порядок ночи)
│   └── handlers/         Реализации способностей по категориям
│       ├── townsfolk.ts
│       ├── outsiders.ts
│       ├── minions.ts
│       └── demons.ts
├── game/
│   ├── setup.ts          Раздача ролей по составу
│   ├── engine.ts         Главный цикл: night → day → win check
│   ├── night.ts          Прогон ночи в заданном порядке
│   ├── day.ts            Обсуждение (упрощённо), номинации, казнь
│   └── win.ts            Проверка победных условий
├── ai/
│   ├── brain.ts          Базовый интерфейс AI и `SimpleBrain`
│   └── suspicion.ts      Матрица подозрений (для smart-режима)
├── stats/
│   └── collector.ts      Сбор статистики по батчу
└── cli.ts                Входная точка
```

## Результаты v0.1 (baseline, случайный AI, 10 игроков × 1000 партий/демон)

```
vecna        good: 17.5%  evil: 82.5%
mindflayer   good: 53.2%  evil: 46.8%
whatsit      good:  4.9%  evil: 95.1%
```

Интерпретация:
- **Mind Flayer** близок к эталонному `~50/50`, что идеально для пассивного AI.
- **Vecna** слишком быстр: 4 смерти по меткам копятся за ~8 дней при отсутствии активных защит.
- **Mr. Whatsit** радикально силён в модели, где добрые почти не голосуют: конверсии копятся быстрее, чем Демона ловят.

Это ожидаемо для v0.1: боты не используют информацию. Ждём v0.2 (smart AI) — тогда
инфо-роли и Хоппер/Стив начнут работать, и баланс выровняется.

## План развития

- **v0.1** ✅ все 25 ролей, случайный AI, CLI-отчёт по winrate.
- **v0.2**: байесовские подозрения у добрых — реакция на ночную инфу, защита помеченных,
  координация злых (не голосуют против демона, прикрывают его клеймы).
- **v0.3**: блеф, социальное давление, неправдивые заявления, анализ публичных роль-клеймов.
- **v0.4**: веб-дашборд с графиками (Trace-viewer для одной партии + сводная аналитика).
- **v0.5**: экспорт `protocol.json` для интеграции с цифровым гримуаром.
