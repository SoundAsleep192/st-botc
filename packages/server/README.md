# @botc/server — онлайн-лобби (заглушка)

Сервер для «Jackbox-style» комнат: ведущий создаёт игру, игроки подключаются по коду/QR.

## Планируемый стек

- **Bun + WebSocket** — нативные WS в Bun, без зависимостей.
- **@botc/core** — общая модель игры с клиентами.
- **Zod** — валидация входящих сообщений.
- **Простые in-memory комнаты** — для первой версии; позже Redis/SQLite для устойчивости.

## Протокол (черновик)

```
ws://host/room/:code

Клиент → Сервер:
  { type: "join", name, role: "player" | "storyteller" }
  { type: "vote", nomineeId }
  { type: "night-action", actorId, targetId }
  { type: "claim", role }

Сервер → Клиент:
  { type: "state", state: PublicGameState }
  { type: "private", info: PrivateNightInfo }
  { type: "prompt", question, options }
```

## MVP scope

1. Создание комнаты ведущим → получение 4-значного кода.
2. Подключение игроков по коду (имя + слот).
3. Раздача ролей: скрытая рассылка каждому игроку его роли.
4. Ночные приватные окна: ведущий помечает цель, игрок видит инфу.
5. Голосование: все поднимают руку через UI, результат виден сразу.
6. Лог партии доступен после окончания.

## Что не входит в MVP

- Авторизация / аккаунты — анонимный вход по коду.
- Голос/видео — используется любой внешний VoIP (Discord, Zoom).
- Хранение истории партий между деплоями.
